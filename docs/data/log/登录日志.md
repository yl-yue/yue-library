# 登录日志

　　登录日志通过 `LoginLogService` 手动调用记录，适用于登录成功/失败等场景。

## 1. 手动调用记录

在登录逻辑中调用 `LoginLogService.recordLogin()`：

```java
@Resource
private LoginLogService loginLogService;

public Result<?> login(String username, String password) {
    boolean success = validatePassword(username, password);
    String msg = success ? "登录成功" : "密码错误";
    loginLogService.recordLogin(username, "127.0.0.1", success, msg, "{\"username\":\"" + username + "\"}");
    if (success) {
        return R.success();
    }
    return R.errorPrompt(msg);
}
```

### 方法签名

```java
void recordLogin(String username, String ip, boolean success, String msg, String requestParam)
```

| 参数 | 类型 | 说明 |
|------|------|------|
| username | String | 登录账号（可能是 username/phone/email） |
| ip | String | 登录 IP |
| success | boolean | 是否登录成功 |
| msg | String | 登录消息（如"登录成功"、"密码错误"） |
| requestParam | String | 请求参数 JSON（会自动脱敏） |

### 自动填充字段

以下字段由 MyBatis-Plus 自动填充机制处理，无需手动赋值：
- `create_user`：创建人（用户昵称/人名）
- `create_user_id`：创建人 ID
- `create_time`：创建时间
- `tenant_sys_id`：系统租户 ID
- `tenant_co_id`：企业租户 ID

> 登录失败时，`create_user` 等字段可能为空（因为用户未认证）。

## 2. 登录日志数据表

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT | 有序主键 |
| username | VARCHAR(64) | 登录账号 |
| ip | VARCHAR(64) | 登录 IP |
| login_time | BIGINT | 登录时间 |
| status | TINYINT | 登录状态（0=失败, 1=成功） |
| msg | VARCHAR(512) | 登录消息 |
| request_param | TEXT | 请求参数（脱敏后 JSON） |
| create_user | VARCHAR(64) | 创建人 |
| create_user_id | BIGINT | 创建人 ID |
| create_time | BIGINT | 创建时间 |
| tenant_sys_id | VARCHAR(64) | 系统租户 ID |
| tenant_co_id | VARCHAR(64) | 企业租户 ID |
