## 快速开始 <!-- {docsify-ignore} -->
　　data-log模块提供基于注解的日志自动记录能力，支持操作日志与登录日志两种类型，特性如下：
- **注解驱动**：只需在方法上添加 `@Log` 注解，即可自动记录操作日志
- **操作类型自动推断**：从 HTTP Method 自动推断操作类型（POST→C, GET→R, PUT/PATCH→U, DELETE→D）
- **请求参数与响应结果自动捕获**：兼容 JSON 与非 JSON 格式
- **数据脱敏**：密码字段自动置空，支持 SPI 自定义脱敏策略
- **异步记录**：默认异步记录日志，不阻塞主流程
- **异常隔离**：日志记录失败绝不影响业务主流程
- **自动建表**：启动时自动创建日志表（可配置关闭）
- **自动归档**：定时归档历史日志数据（默认 3 个月或超百万条）
- **微服务支持**：通过 `LogStorageProvider` SPI 接口实现可插拔持久化，内置 HTTP 转发模式
- **请求日志拦截器**：自动注入 traceId、打印请求信息，默认启用
- **查询接口**：提供分页查询与详情查询能力，供业务管理后台审计使用

### 引入模块
`yue-library`作为父项目，在`pom.xml`文件中添加：
``` pom
<dependencies>
	<dependency>
		<groupId>ai.ylyue</groupId>
		<artifactId>yue-library-data-log</artifactId>
	</dependency>
</dependencies>
```

> `yue-library-data-mybatis` 和 `yue-library-web` 已通过依赖传递自动引入，无需额外声明。

### 配置数据源
日志模块依赖 `data-mybatis` 的数据源配置，请参考 [👉data-mybatis 快速开始](data/mybatis/快速开始.md)

### 启用yue-library提供的默认配置
在 `application.yml` 或 `bootstrap.yml` 文件中添加如下配置：
```yml
spring:
  profiles:
    group:
      "yue": "yue-library-web,yue-library-data-mybatis,yue-library-data-redis"
    active: yue
    ...
```

### 配置 logback
日志模块内置标准 logback 配置模板，包含 traceId 输出、plumelog 日志聚合、文件输出等能力。在 `application.yml` 中添加一行配置即可启用：
```yml
logging:
  config: classpath:yue-logback-spring.xml
```

> Spring Boot 的日志系统在启动早期初始化，无法通过代码层面自动设置 `logging.config`，因此需要项目自行声明。

**环境差异化配置**：
```yml
# application-test.yml（测试环境）
spring:
  profiles:
    group:
      log: plumelog
plumelog:
  redisHost: 127.0.0.1
  redisPort: 6379
  redisAuth: xxx
  redisDb: 0
  env: test

# application-prod.yml（生产环境）
spring:
  profiles:
    group:
      log: file,plumelog
plumelog:
  redisHost: prod-redis
  redisPort: 6379
  redisAuth: xxx
  redisDb: 0
  env: prod
```

| Profile | 说明 | 适用环境 |
|---------|------|----------|
| 默认（无额外 Profile） | 仅控制台输出 | 本地开发 |
| `plumelog` | 控制台 + plumelog 日志聚合 | 测试/生产 |
| `file,plumelog` | 控制台 + 文件 + plumelog | 生产 |

### 持久化模式
日志模块默认使用 HTTP 转发模式（`mode=http`），适用于微服务架构。

**单体架构**（直连写库）：
```yml
yue:
  data:
    log:
      mode: direct
```

**微服务架构**（HTTP 转发到日志服务）：
```yml
yue:
  data:
    log:
      http:
        base-url: http://log-service:8080  # 日志服务地址
```

> 详细的微服务配置说明请参考 [👉微服务日志](data/log/微服务日志.md)

### 启用查询接口（可选）
如需在业务管理后台查询审计日志，需显式开启查询接口：
```yml
yue:
  data:
    log:
      query-api:
        enabled: true
```

### 日志表自动创建
日志模块默认在应用启动时自动创建 `log_login`、`log_oper` 及对应的归档表，无需手动建表。如需关闭自动建表：
```yml
yue:
  data:
    log:
      auto-create-table: false
```

> 自动建表仅在 `mode=direct`（直连写库）模式下生效，HTTP 转发模式下由日志服务端负责建表。

### 配置项一览

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `yue.data.log.enabled` | true | 全局开关 |
| `yue.data.log.mode` | http | 持久化模式：http / direct |
| `yue.data.log.async` | true | 是否异步记录 |
| `yue.data.log.mask-enabled` | true | 是否启用脱敏 |
| `yue.data.log.auto-create-table` | true | 是否自动建表 |
| `yue.data.log.archive-enabled` | true | 是否启用归档 |
| `yue.data.log.archive-months` | 3 | 归档时间阈值（月） |
| `yue.data.log.archive-count` | 1000000 | 归档数量阈值 |
| `yue.data.log.max-param-size` | 10240 | 请求参数最大长度（字节） |
| `yue.data.log.max-response-size` | 51200 | 响应结果最大长度（字节） |
| `yue.data.log.http.base-url` | - | 日志服务地址（mode=http 时必填） |
| `yue.data.log.request-log.enabled` | true | 是否启用请求日志拦截器 |
| `yue.data.log.request-log.exclude-paths` | /actuator-security/** | 排除路径（Ant 风格） |
| `yue.data.log.storage-api.enabled` | false | 是否启用持久化接口 |
| `yue.data.log.query-api.enabled` | false | 是否启用查询接口 |
