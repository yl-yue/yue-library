# 微服务日志转发

　　在微服务架构下，日志模块提供 HTTP 转发模式，将日志数据统一收敛到日志服务。

## 1. 架构方案

```
┌─────────────────────────────────────────────────────┐
│              data-log 框架模块（所有微服务可用）          │
│                                                       │
│  @Log 注解  →  LogAspect 切面  →  数据组装与脱敏        │
│                    │                                  │
│                    ▼                                  │
│          LogStorageProvider (SPI 接口)                 │
│                    │                                  │
│         ┌─────────┴──────────┐                       │
│         ▼                    ▼                        │
│  DefaultLogStorage    HttpLogStorage                  │
│  Provider (直连写库)   Provider (HTTP转发)             │
│  mode=direct          mode=http（默认）               │
└─────────────────────────────────────────────────────┘
```

## 2. 三种持久化模式

| 模式 | 适用场景 | 配置 | 说明 |
|------|----------|------|------|
| **HTTP 转发**（默认） | 微服务架构（非日志服务） | 默认（`mode=http`）+ 配置日志服务地址 | 通过 Forest HTTP 客户端转发至日志服务 |
| **直连写库** | 单体架构 / 日志服务本身 | `mode=direct` | 通过 MyBatis-Plus Mapper 直接写库，自动建表 |
| **自定义 SPI** | 需 MQ 等其他中间件转发 | 实现 `LogStorageProvider` 接口并注册为 Bean | 用户完全自定义持久化逻辑 |

> **默认模式选择原则**：默认为 HTTP 转发模式（`mode=http`），因为微服务是更常见的部署形态。微服务忘记配置 HTTP 转发时，若默认直连写库会静默建表污染业务数据库——这种失败模式比"缺少配置而 WARN 提示"危险得多。单体/日志服务只需一行 `mode=direct` 显式声明。

### 持久化策略选择优先级

```
用户自定义 LogStorageProvider Bean > mode=http → HttpLogStorageProvider（默认） > mode=direct → DefaultLogStorageProvider
```

## 3. HTTP 转发模式配置

### 发送端（非日志服务）

```yml
yue:
  data:
    log:
      http:
        base-url: http://log-service:8080  # 日志服务地址（必填）
```

> `mode` 默认为 `http`，无需显式配置。仅需配置 `http.base-url`。

### 接收端（日志服务）

```yml
yue:
  data:
    log:
      mode: direct  # 日志服务本身使用直连写库
      storage-api:
        enabled: true  # 启用持久化接口，供其他微服务 HTTP 调用
```

> 持久化接口使用 `lan` 安全前缀（`/lan/v1/log/`），仅限局域网内部微服务调用。

### 启动时安全校验

| 场景 | 行为 |
|------|------|
| `mode=http` + `base-url` 已配置 + Forest 已引入 | 正常 HTTP 转发 |
| `mode=http` + `base-url` 未配置 | WARN: "HTTP 模式未配置 base-url，日志记录将不生效" |
| `mode=http` + Forest 未引入 | WARN: "HTTP 模式需要 Forest 依赖，请引入 forest-spring-boot3-starter" |
| `mode=direct` | 正常建表 + 直连写库 |

## 4. 自定义持久化策略

实现 `LogStorageProvider` SPI 接口并注册为 Spring Bean：

```java
@Component
public class MqLogStorageProvider implements LogStorageProvider {

    @Resource
    private RocketMQTemplate rocketMQTemplate;

    @Override
    public void storeLoginLog(LoginLogEntity entity) {
        rocketMQTemplate.convertAndSend("log-login-topic", entity);
    }

    @Override
    public void storeOperLog(OperLogEntity entity) {
        rocketMQTemplate.convertAndSend("log-oper-topic", entity);
    }
}
```

> 注册自定义 `LogStorageProvider` Bean 后，将优先使用自定义实现，内置的直连写库和 HTTP 转发实现不再生效。

## 5. SPI 接口

### LogUserProvider

获取当前登录账号，业务项目需实现此接口：

```java
@Component
public class CustomLogUserProvider implements LogUserProvider {
    @Override
    public String getUsername() {
        return StpUtil.getLoginIdAsString();
    }
}
```
