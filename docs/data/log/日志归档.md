# 日志归档

　　日志模块内置自动归档能力，将历史日志数据迁移到归档表，保证主表查询性能。

## 1. 归档机制

归档任务通过 `@Scheduled` 定时任务执行，满足以下任一条件即触发归档：
- 数据创建时间超过归档时间阈值（默认 3 个月）
- 数据量超过归档数量阈值（默认 100 万条）

### 归档流程

```
[@Scheduled 定时任务] → [检查数据量与时间] → [按 ID 范围批量归档] → [事务内 INSERT + DELETE]
```

归档操作在事务内完成，保证数据一致性：
1. 查询满足条件的 ID 范围：`SELECT MIN(id), MAX(id) FROM log WHERE create_time < ?`
2. 归档：`INSERT INTO archive SELECT * FROM log WHERE id BETWEEN ? AND ?`
3. 删除：`DELETE FROM log WHERE id BETWEEN ? AND ?`

### 归档表

| 归档表 | 对应主表 | 说明 |
|--------|----------|------|
| log_login_archive | log_login | 登录日志归档表 |
| log_oper_archive | log_oper | 操作日志归档表 |

> 归档表结构与主表完全一致，归档数据可从归档表查询。

## 2. 配置项

```yml
yue:
  data:
    log:
      archive-enabled: true       # 是否启用归档（默认 true）
      archive-months: 3           # 归档时间阈值（月，默认 3）
      archive-count: 1000000      # 归档数量阈值（默认 100 万）
```

## 3. 定时任务时间

| 任务 | 执行时间 | 说明 |
|------|----------|------|
| 登录日志归档 | 每天凌晨 2:00 | `cron = "0 0 2 * * ?"` |
| 操作日志归档 | 每天凌晨 3:00 | `cron = "0 0 3 * * ?"` |

> 归档任务使用 `@Scheduled` 单线程执行，避免并发归档。
