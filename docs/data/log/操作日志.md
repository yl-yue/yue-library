# 操作日志

　　操作日志通过 `@Log` 注解自动记录，支持以下核心能力：

## 1. 注解驱动自动记录

在需要记录操作日志的 Controller 方法上添加 `@Log` 注解：

```java
@Log(title = "新增用户", description = "管理员新增用户账号")
@PostMapping("/user/insert")
public Result<?> insert(UserIPO userIPO) {
    userService.insert(userIPO);
    return R.success();
}
```

### 注解属性说明

| 属性 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `type` | LogTypeEnum | 否 | OPER | 日志类型（OPER=操作日志, LOGIN=登录日志） |
| `title` | String | 是 | - | 操作标题 |
| `description` | String | 否 | "" | 操作描述 |
| `operType` | DbCrudEnum | 否 | R | 操作类型，默认从 HTTP Method 自动推断 |
| `bizType` | String | 否 | "" | 业务分类（如 IMPORT/EXPORT） |

### 操作类型自动推断

`operType` 默认从 HTTP Method 自动推断，无需手动指定：

| HTTP Method | 推断操作类型 | 说明 |
|-------------|-------------|------|
| POST | C（新增） | 创建类操作 |
| GET | R（查询） | 查询类操作 |
| PUT / PATCH | U（更新） | 修改类操作 |
| DELETE | D（删除） | 删除类操作 |

如需覆盖推断值，可显式指定 `operType`：

```java
@Log(title = "导入数据", operType = DbCrudEnum.C, bizType = "IMPORT")
@PostMapping("/data/import")
public Result<?> importData(String data) {
    return R.success();
}
```

### 操作状态自动推断

操作状态从 `Result.flag` 自动推断：
- `flag=true` → status=1（成功）
- `flag=false` → status=0（失败）
- 方法抛异常 → status=0（失败）

## 2. 请求参数与响应结果

- **请求参数**：通过 `ServletUtils.getParamToJson()` 自动捕获，脱敏后存储
- **响应结果**：仅存储 JSON 格式响应，非 JSON 或超大响应（> 50KB）不存储
- **参数截断**：请求参数超过 10KB 时自动截断，保留前 10KB

## 3. 数据脱敏

默认脱敏策略对密码类字段置空处理：
- `password`、`pwd`、`passwd`、`secret`、`accesstoken`、`refreshtoken` 等字段自动置空

可通过 SPI 接口 `LogMaskStrategy` 实现自定义脱敏逻辑：

```java
@Component
public class CustomLogMaskStrategy implements LogMaskStrategy {
    @Override
    public String mask(String param) {
        // 自定义脱敏逻辑
        return maskedParam;
    }
}
```

配置项：
```yml
yue:
  data:
    log:
      mask-enabled: true  # 是否启用脱敏（默认 true）
```
